# Web Brief

本文件用于给不能直接读取仓库的 web 模型复制粘贴。

## 项目摘要

- 项目：遗迹搬运队 / Ruin Crew
- 技术：Godot 4.3 + GDScript
- 类型：3D 俯视角合作探险
- 平台：PC（Windows）

## 当前仓库真实状态

- 仓库目前以文档和目录骨架为主
- 当前未确认存在已配置的主场景
- 当前未确认存在已注册的 Autoload
- 文档进度领先于运行时代码

## 这是一个多 AI 协作项目

不同 AI 的典型职责：

- ChatGPT / Claude web：设计讨论、方案比较、文案、流程审阅
- Codex / Claude Code：仓库探索、实现、修复、测试、文档同步
- Kimi：批量数据、模板、清洗

## web 模型应该做什么

- 帮忙做方案比较
- 审阅工作流和文档逻辑
- 生成清晰的功能说明或需求卡
- 输出可交给仓库型 agent 的结构化规格

## web 模型不要做什么

- 不要声称你已经修改了仓库文件
- 不要假设 `source/main.tscn`、`GameState`、`EventBus` 已存在
- 不要把 GDD/TDD 中的设想直接当作代码事实
- 不要把 3D 项目默认当成纯 2D 像素项目

## 使用方式

把本文件与以下内容一起提供给 web 模型：

1. 当前 Sprint 卡片（来自 `docs/tasks.md`）
2. 必要的 `docs/gdd.md` / `docs/tdd.md` 摘录
3. 如有需要，再补 `docs/ai_handoff.md` 最近一条记录

## 建议提示语

```text
请基于以下仓库简报和当前任务卡工作。
如果你的判断依赖于仓库真实状态，请明确区分“已验证事实”和“设计设想”。
输出请尽量结构化，方便下一棒 Codex / Claude Code 直接落地。
```
