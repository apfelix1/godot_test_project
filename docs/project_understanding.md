# 项目全局理解（2026-03）

## 1) 当前项目状态

这是一个 **Godot 4.3 + GDScript** 的早期项目骨架，文档体系完整，但代码与场景内容基本还未展开。

- 已明确：项目定位是 3D 俯视角合作探险（“遗迹搬运队 / Ruin Crew”）
- 已搭好：目录结构、开发流程、规范约束、任务管理模板
- 未展开：核心玩法系统、具体关卡、角色与 UI 实装

## 2) 目标游戏与玩法方向（来自 GDD）

项目目标是做一个偏轻量、偏欢乐协作的探险闯关体验：

- 核心是“分工探路、开机关、搬运宝物、撤离”
- 强调局内协作混乱乐趣 + 局外轻养成承接
- 平台目标为 PC（Windows）

> 目前 GDD 中许多细节仍是“待定”，属于方向确定、细则待落地阶段。

## 3) 技术架构理解（来自 TDD + 架构文档）

### 3.1 场景层级（规划态）

主场景规划为：

- `Main`
  - `World`（含关卡、实体、导航）
  - `CameraRig`（俯视角相机）
  - `UI`（HUD + 菜单）
  - `Systems`（纯逻辑系统节点）

### 3.2 全局单例（规划态）

- `GameState`：全局状态/存档/场景切换
- `EventBus`：跨系统广播事件

### 3.3 通信原则（已定义）

- 广播事件：走 `EventBus`
- 一对一命令：直接引用调用
- 父子协作：子→父用信号，父→子用方法
